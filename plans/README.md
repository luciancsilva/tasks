## Findings considered and rejected

Vetted against source and dismissed (so they aren't re-audited):

- **OIDC auto-admin no-op** (run-1 finding) — REJECTED 2026-07-12: the
  `User.afterCreate` hook (`models/user.js:257-276`) creates the Role row in
  the same transaction, so `Role.update` in
  `provisioningService.js:94-99` matches a row and works.
- **CSRF missing** — rejected: `lusca.csrf` applied to session-authed
  state-changing routes (`backend/app.js:184-193`).
- **SQL injection** — rejected: parameterized Sequelize throughout runtime code.
- **Markdown XSS** — rejected: `react-markdown@10` sanitizes by default, no
  `rehype-raw`.
- **Mass assignment** — rejected: task create and profile update allowlist fields.
- **moment vs date-fns duplication** — rejected as bundle issue: moment-timezone
  backend-only, date-fns frontend-only.
- **feature-flags stale flags** — rejected: env-boolean capability toggles, not
  dead branching.
- **Recurring drag-drop rename/delete bugs (#1249/#1252/#1253)** — verified
  fixed: virtual-occurrence expansion is gated to `type=upcoming&groupBy=day`,
  so Calendar/Kanban/Eisenhower never receive id-colliding virtual rows.
- **`TaskList` missing React.memo** — rejected: `TaskItem` is memoized
  (`TaskItem.tsx:538`); per-row re-renders already cheap.
- **SQLite pragma tuning** — rejected: WAL, synchronous=NORMAL, busy_timeout,
  64MB cache, mmap already set (`models/index.js:22-51`).
- **`tasks` core index coverage** — rejected: `status`, `due_date`,
  `project_id`, `parent_task_id`, `user_id` all indexed (`models/task.js:236-286`);
  only `area_id` is missing (listed above).
